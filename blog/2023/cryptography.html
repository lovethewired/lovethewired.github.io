1.7 Symmetric and asymmetric ciphers

1.7.1 Symmetric ciphers

Symmetric ciphers are when both parties share the same (or symmetric) knowledge of the secret key kkk. E.g. Bob and Alice both encrypt and decrypt a message using the same secret key.

The encryption function eee takes a secret key kkk in the space of all possible keys K\mathcal KK and transforms a message mmm from all possible messages M\mathcal MM to a ciphertext ccc belonging to the space of all possible ciphertexts C\mathcal CC.

e:K×M→Ce: \mathcal K \times \mathcal M \to \mathcal Ce:K×M→C

Decrypting takes the same key kkk and maps the ciphertext ccc back to a message mmm.

d:K×C→Md: \mathcal K \times \mathcal C \to \mathcal Md:K×C→M

The decryption function should map back to the preimage of the encryption one-to-one.

d(k,e(k,m))=m∀k∈K,m∈Md(k, e(k, m)) = m \quad \forall k \in \mathcal K, m \in \mathcal Md(k,e(k,m))=m∀k∈K,m∈M

The following notation will be used.

ek:M→Cdk:C→Mdk(ek(m))=me_k: \mathcal M \to \mathcal C \qquad d_k: \mathcal C \to \mathcal M \qquad d_k(e_k(m)) = mek​:M→Cdk​:C→Mdk​(ek​(m))=m

Kerckhoff's principle. The security of a crypto system should only depend on the secrecy of the key and not the secrecy of the encryption algorithm itself.

A few desirable properties are:

  1. ek(m)e_k(m)ek​(m) should be easy to compute.
  2. dk(m)d_k(m)dk​(m) should be easy to compute.
  3. Given c1,c2,…,cn∈Cc_1, c_2, \ldots, c_n \in \mathcal Cc1​,c2​,…,cn​∈C it should be very difficult to compute any plaintext dk(c1),…,dk(cn)d_k(c_1), \ldots, d_k(c_n)dk​(c1​),…,dk​(cn​) without knowledge of kkk.

An additional stronger property would be:

  1. Security against a chosen plaintext attack: Given any number of pairs of plaintexts and their corresponding ciphertexts, (m1,c1),(m2,c2),…,(mn,cn)(m_1, c_1), (m_2, c_2), \ldots, (m_n, c_n)(m1​,c1​),(m2​,c2​),…,(mn​,cn​), it must also be difficult to decrypt any ciphertext ccc that is not in the list without knowledge of kkk.

Any substitution cipher does not satisfy property 4, since any plaintext-ciphertext pair reveals a lot about the encryption table.

Definition. An encoding scheme transforms data, e.g. text to numbers, and consists of an encoding and decoding function which are publicly known, fast, and easy to compute.

1.7.3 Symmetric encryption of encoded blocks

Any message m∈Mm \in \mathcal Mm∈M can be seen as a bit string of fixed length BBB (blocksize). The encryption and decryption functions can then transform plaintext message blocks of fixed length to ciphertext blocks of fixed––but not necessarily the same––length. If a message is too short to fill out the block, it can be padded.

K={k∈Z:0≤k<2Bk}M={m∈Z:0≤m<2Bm}C={c∈Z:0≤c<2Bc}\begin{aligned} \mathcal K &= \{ k \in \mathbb Z: 0 \le k \lt 2^{B_k}\} \\ \mathcal M &= \{ m \in \mathbb Z: 0 \le m \lt 2^{B_m}\} \\ \mathcal C &= \{ c \in \mathbb Z: 0 \le c \lt 2^{B_c}\} \end{aligned}KMC​={k∈Z:0≤k<2Bk​}={m∈Z:0≤m<2Bm​}={c∈Z:0≤c<2Bc​}​

A key block size BkB_kBk​ should be chosen large enough such that a brute-force attack (exhaustive search) is infeasible.

Example 1.35. (Vernam's one-time pad).

Using the XOR operation ⊕\oplus⊕ a one-time pad is generated by selecting a key kkk which is as long as the message itself.

ek(m)=k⊕mdk(c)=k⊕ce_k(m) = k \oplus m \qquad d_k(c) = k \oplus cek​(m)=k⊕mdk​(c)=k⊕c

If kkk is random, then the scheme is provably secure, as long as the key is only used once. The biggest problem with this cipher is the key length requirement. It is possible to use a small length key kkk to generate a one-time pad through a pseudorandom number generator seeded with the key RkR_kRk​: Rk(1),Rk(2),…,Rk(BM)R_k(1), R_k(2), \ldots, R_k(B_M)Rk​(1),Rk​(2),…,Rk​(BM​). This idea lies at the core for most practical symmetric ciphers of the Data Encryption Standard (DES) and the Advanced Encryption Standard (AES). However, the security guarantees are unclear. It is more desirable to create cipher schemes whose inversion relied on a well-known mathematical problem believed to be difficult to solve.

1.7.6 Asymmetric ciphers make a first appearance
